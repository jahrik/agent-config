# Python Skill

## Conventions

- **Formatting & Linting:** `ruff` (run via `uv run ruff check` and `uv run ruff format`)
- **Package & Dependency Management:** `uv` (using `pyproject.toml` and `uv.lock`)
- **Type hints:** preferred, not required for scripts
- **Python version:** 3.11+ preferred
- **Venvs:** per-project managed locally via `uv` in `.venv/`

## Project Structure

```
src/              # or top-level package/module
tests/
pyproject.toml    # project config and ruff definitions
uv.lock           # pinned locks generated by uv
.github/workflows/
README.md
AGENTS.md
```

## Common Commands

```bash
uv init           # Initialize a new project with pyproject.toml
uv sync           # Install dependencies and sync virtual environment
uv add <pkg>      # Add a package to dependencies
uv add --dev <pkg># Add a package to development dependencies
uv run ruff check . # Lint the codebase
uv run ruff format . # Format the codebase
uv run pytest     # Run tests
```

## CI Pattern

GitHub Actions running `uv run ruff check .` and tests on push to `main`/`master` (using the `astral-sh/setup-uv` action).
