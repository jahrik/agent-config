# Python Skill

## Conventions

- **Package & dependency management:** `uv` (`pyproject.toml` + `uv.lock`); per-project `.venv/`.
- **Formatting & linting:** `ruff` — `uv run ruff format` and `uv run ruff check`. Ruff config lives in `[tool.ruff]` in `pyproject.toml`; keep it there, not in separate dotfiles.
- **Type checking:** `ty` (Astral's checker) via `uv run ty check`. Type hints are expected on library code; looser for one-off scripts.
- **Testing:** `pytest` (see below).
- **Local gate:** `pre-commit` wires the above together (see below).
- **Python version:** 3.11+ supported; repos currently target 3.13.

## Project Structure

```
src/              # importable package(s); prefer a src layout over top-level modules
tests/
pyproject.toml    # single source of truth: project + [tool.ruff] + [tool.ty] + pytest config
uv.lock           # pinned lockfile, generated by uv — commit it
.pre-commit-config.yaml
.github/workflows/
README.md
AGENTS.md
```

## Common Commands

```bash
uv init             # scaffold a new project (src layout)
uv sync             # install deps + sync the venv
uv add <pkg>        # add a runtime dependency
uv add --dev <pkg>  # add a dev dependency
uv run ruff format .              # format
uv run ruff check .               # lint (add --fix to auto-fix)
uv run ty check                   # type check
uv run pytest                     # run tests
uvx pre-commit run --all-files    # run every gate locally, as CI would
```

## Testing Conventions

- Tests live in `tests/`, mirroring the package layout; use plain `pytest` (fixtures over setup/teardown classes).
- **No test reaches the network or a live external service.** Mock `subprocess`, HTTP clients, and any `gh`/API calls (`pytest-mock`, `monkeypatch`) so the suite is fully hermetic and deterministic.
- Aim for high coverage; some repos enforce a hard bar (e.g. 100%) in `pyproject.toml` / CI — honor the target the repo already sets.

## pre-commit

`pre-commit` is the local quality gate — it runs the same checks CI does before a commit lands. Standardize on a `.pre-commit-config.yaml` that runs `ruff` (lint + format), `ty`, and the secret-scan/test hooks the repo uses. Install once per clone with `uv run pre-commit install`; run across the tree with `uvx pre-commit run --all-files`.

## CI Pattern

GitHub Actions using `astral-sh/setup-uv`, running the same gates on push / PR: `uv run ruff check .`, `uv run ty check`, and `uv run pytest`. CI and pre-commit should run the same commands, so green-locally means green-in-CI. Publishing/packaging specifics (`uv build`, `uv publish`, `uv tool install`) belong in each repo's `AGENTS.md`, not this portable base.
